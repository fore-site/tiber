Tiber is a notification delivery platform that doesn't just send messages — it decides _when_ to send them, _how urgent_ they are, and _what they should say_, using a mix of trained ML models and an LLM layer, all behind production-grade reliability infrastructure (queues, retries, dead-letter handling, idempotency, provider failover). It includes a self-built data simulator that generates realistic synthetic user engagement data, and that simulator is itself an engineered, documented component, not a throwaway script.

The system demonstrates: async distributed processing, provider/channel abstraction designed for extensibility, ML model training/evaluation with honest metrics, LLM integration with a clear boundary between "ML decision" and "AI-generated content," observability, and a deployed, demoable product.
